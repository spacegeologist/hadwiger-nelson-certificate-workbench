# 2026-06-18 Phase 5

## Goal

Push past Phase 4's weak "remove 3-8 edges via DRAT core" result toward something
genuinely defensible: (a) a principled deletion-minimization engine, (b) a
*rigorous* (exact, non-floating-point) geometry certificate, and (c) an honest
assessment of what is and is not reachable by this method.

## Literature frontier (grounded 2026-06-18)

The smallest known 5-chromatic unit-distance graph in the plane is **Jaan Parts
(2020): 509 vertices, 2442 edges** (arXiv:2010.12665). History: de Grey 2018
(1581 v / 7877 e) → Heule (553 → 517 v) → Parts (509 v / 2442 e). CNP-SAT's
smallest non-4-colorable graph is the **510-vertex / 2504-edge** graph.

## A deletion lower bound: 509 is unreachable from the 510 graph

**Claim.** Pure deletion (removing edges and/or vertices) starting from the
510-vertex graph cannot produce a non-4-colorable graph with fewer than 510
vertices.

**Proof.** The 510 graph `G` is *vertex-critical*: deleting any vertex makes it
4-colorable. We verified this exhaustively — `scripts/check_vertex_critical.py`
confirms all 510 single-vertex deletions are 4-colorable (116.6 s). Phase 3
observed the same for 517.
So for every vertex `v`, `G - v` is 4-colorable. Let `G'` be any edge-subgraph of
`G` that is still non-4-colorable, and suppose some vertex `v` has degree `<= 1`
in `G'`. Then `G' - v` is an edge-subgraph of `G - v`, which is 4-colorable;
hence `G' - v` is 4-colorable, and since `v` has degree `<= 1` it can always be
colored last, so `G'` is 4-colorable — a contradiction. Therefore no vertex of a
still-non-4-colorable edge-subgraph can be pruned, and the vertex count is pinned
at 510. ∎

**Consequence.** Edge/vertex deletion from CNP-SAT cannot beat Parts' 509-vertex
record. Beating the record requires a *different vertex set* (genuine
construction/search, as Parts and Heule did), which is out of scope for a
deletion-only pipeline. We do not claim a new bound, and we do not claim a record.

## What we built

- `hnp/minimize.py`: incremental SAT criticality reducer.
  - Color-symmetry breaking by pinning a triangle to 3 distinct colors.
  - Per-edge selector literals so one incremental solver answers "still UNSAT
    without this edge set?" through assumptions, reusing learned clauses.
  - Per-solve conflict budget; an edge is removed only when its removal is
    *proved* UNSAT within budget, so the output is always still non-4-colorable.
  - `prune_low_degree_vertices`: safe degree-`<=1` pruning (never fires on a
    vertex-critical input, consistent with the bound above).
- `hnp/exact_embedding.py` + `scripts/verify_embedding_exact.py`: exact symbolic
  unit-distance verification in the algebraic field `Q(sqrt 3, sqrt 5, sqrt 11)`.
  Each edge passes only when `(dx^2 + dy^2 - 1)` reduces identically to 0.
- `scripts/minimize_graph.py`: end-to-end runner (minimize → prune → independent
  re-verify on a fresh solver → exact/float geometry → JSON report).

## Exact geometry certificates (new, rigorous)

Replacing the `|dist - 1| < 1e-9` tolerance check with an exact symbolic proof.
All four CNP-SAT graphs verify exactly:

| graph | vertices | edges | exact-unit result | seconds |
| --- | ---: | ---: | --- | ---: |
| 510 | 510 | 2504 | all edges exactly unit | 2.29 |
| 517 | 517 | 2579 | all edges exactly unit | 2.09 |
| 529 | 529 | 2670 | all edges exactly unit | 2.46 |
| 553 | 553 | 2722 | all edges exactly unit | 3.42 |

This upgrades the geometry side of the certificate from a numerical sanity check
to a proof.

## Non-4-colorability re-confirmation

`510.edge` solved to UNSAT on a fresh CaDiCaL with no symmetry breaking in 83.1 s
(2040 vars, 13586 clauses). With the triangle pinned (vertices 1,2,6 → colors
0,1,2) the same UNSAT is reached in ~15 s and incrementally far faster.

## Edge minimization (best-effort, order-dependent)

Run on `510.edge`, lexicographic order, conflict budget 8000 per solve
(`data/hadwiger-nelson/generated/minimized/510-min.report.json`):

| quantity | value |
| --- | ---: |
| input | 510 v / 2504 e |
| edges removed (proved redundant) | 217 |
| **result** | **510 v / 2287 e** |
| vertices pruned | 0 (as the bound predicts) |
| solves | 2501, total 1323.6 s |
| budget-exceeded (kept conservatively) | 135 |
| independent re-check (fresh CaDiCaL, no SBP) | **UNSAT** in 459.0 s |
| exact geometry of surviving edges | all 2287 exactly unit |

So `510-min` is a verified non-4-colorable unit-distance graph on 510 vertices
with 2287 edges — fewer edges than Parts' 2442 but one more vertex, i.e. a
different point on the (vertices, edges) frontier, **not** a record. The 135
budget-exceeded edges mean 2287 is only an upper bound on what this order can
remove; a larger budget or multiple orders would remove more. Because the graph
is vertex-critical, no amount of further edge removal can drop the vertex count.

contribution, not a new bound.

## DRAT certification

`scripts/certify_drat.py` builds the symmetry-broken coloring CNF, emits a DRUP
proof with pysat's Glucose4, and checks it with drat-trim. (pysat's CaDiCaL
proof logging returned incomplete proofs under preprocessing — "conflict
claimed, but not detected" — so Glucose4's DRUP output, which checks cleanly, is
used.) Each reduced graph's certificate verifies; CNFs and certificate summaries
are committed, the multi-MB proofs are regenerable and git-ignored.

## Honest positioning

- No new lower bound; the plane's chromatic number bound stays at 5.
- No record-size graph; 509 is provably unreachable by deletion here.
- Genuine, defensible contributions: an open incremental criticality-reduction
  tool, an *exact* (proof-grade) geometry verifier for the CNP-SAT family, and a
  clean deletion lower-bound argument. Best framed as a reproducibility +
  verification computational note, not a breakthrough.
